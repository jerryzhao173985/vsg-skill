```cpp
#include <vsg/lighting/SoftShadows.h>
#include <vsg/lighting/PercentageCloserSoftShadows.h>
#include <vsg/lighting/Light.h>                        // Light::shadowSettings member
#include <vsg/app/View.h>                              // ViewFeatures, View::create
#include <vsg/state/ViewDependentState.h>              // maxShadowDistance/shadowMapBias/lambda
```

## When to use

Use when you want a `vsg::DirectionalLight` or `vsg::SpotLight` to cast real-time
shadow maps onto the scene. Shadows in VSG are a property of an individual **light**,
not of the scene or the view: you opt a light into shadow casting by assigning a
`ShadowSettings` subclass to its `shadowSettings` member (`include/vsg/lighting/Light.h:34`).
Pick the subclass by quality/cost trade-off:

- `HardShadows` — crisp, cheapest, single-sample shadow edges (`include/vsg/lighting/HardShadows.h:20`).
- `SoftShadows` — fixed-radius penumbra (PCF), `penumbraRadius` controls softness (`include/vsg/lighting/SoftShadows.h:23,26`).
- `PercentageCloserSoftShadows` (PCSS) — physically-plausible contact-hardening penumbra (`include/vsg/lighting/PercentageCloserSoftShadows.h:20`).

Only `DirectionalLight` and `SpotLight` cast shadows — there is no omnidirectional
`PointLight` shadow support; `AmbientLight` never casts. Cross-ref `lighting.md` for
how lights are placed in the scene graph.

## Key API

`ShadowSettings` is a `vsg::Object` subclass; every variant carries a `shadowMapCount`
(the number of cascades / shadow-map slices for that light), defaulting to 1
(`include/vsg/lighting/ShadowSettings.h:23,26`).

```cpp
// create() factories (generated by Inherit<>), with their real ctor signatures:
vsg::HardShadows::create(uint32_t shadowMaps = 1);                                  // HardShadows.h:23
vsg::SoftShadows::create(uint32_t shadowMaps = 1, float penumbraRadius = 0.05f);    // SoftShadows.h:23
vsg::PercentageCloserSoftShadows::create(uint32_t shadowMaps = 1);                  // PercentageCloserSoftShadows.h:23
```

Attach to a light (public member, assigned directly — there is no setter):

```cpp
light->shadowSettings = vsg::HardShadows::create(numShadowMaps);   // Light.h:34
```

`View::create()` defaults its `ViewFeatures` to `RECORD_ALL`, i.e.
`RECORD_LIGHTS | RECORD_SHADOW_MAPS` (`include/vsg/app/View.h:31,38`). Because
`RECORD_SHADOW_MAPS` is on by default, shadow-map recording is active and
`viewer->compile()` auto-sizes the shadow maps from the lights it finds — you do **not**
wire up shadow-map capacity manually: when `RECORD_SHADOW_MAPS` is set, `ViewDependentState` sums each shadow-casting light's `shadowMapCount` and clamps it to the device's `numShadowMapsRange` (`src/vsg/state/ViewDependentState.cpp:230-237,248-253`). (`View::create(camera, scenegraph)` also defaults
to `RECORD_ALL` — `View.h:43`.)

Cascade/quality hints live on `view->viewDependentState` (`include/vsg/app/View.h:87`),
a `vsg::ViewDependentState` (`include/vsg/state/ViewDependentState.h:105`):

| member | default | meaning | cite |
| --- | --- | --- | --- |
| `maxShadowDistance` | `1e8` | far cut-off beyond which shadows stop | `ViewDependentState.h:154` |
| `shadowMapBias` | `0.005` | depth bias to fight shadow acne | `ViewDependentState.h:155` |
| `lambda` | `0.5` | cascade split blend (uniform↔logarithmic) | `ViewDependentState.h:156` |
| `shadowSettingsOverride` | `{}` | per-light override map; `[{}]` entry matches all lights not otherwise matched | `ViewDependentState.h:159` |

`shadowSettingsOverride` is a `std::map<ref_ptr<const Light>, ref_ptr<ShadowSettings>>`;
assigning `shadowSettingsOverride[{}] = settings` overrides the ShadowSettings for every
light not explicitly keyed (`include/vsg/state/ViewDependentState.h:159-160`). The
resolved settings for a given light come from
`getActiveShadowSettings(const Light*)` (`include/vsg/state/ViewDependentState.h:162`).

## Best Practices

- Set `shadowSettings` on the light **before** `viewer->compile()` — compile sizes the
  shadow maps from the lights it discovers (`examples/lighting/vsgshadow/vsgshadow.cpp:538,654`).
- Use one `ShadowSettings` instance shared across all lights that should behave
  identically; the example assigns the same `shadowSettings` ref to multiple
  directional/spot lights (`examples/lighting/vsgshadow/vsgshadow.cpp:538,564,586`).
- Tune cascades on the `ViewDependentState`, not the light: set `maxShadowDistance`,
  `shadowMapBias`, `lambda` after `View::create()`
  (`examples/lighting/vsgshadow/vsgshadow.cpp:628-630`).
- Raise `shadowMapBias` if you see shadow acne; raise `shadowMapCount` (cascades) for
  large outdoor scenes that need shadow detail across depth.
- A hand-wired `View` has **no** auto-headlight, so lit geometry renders black unless you
  add lights (or lit geometry) to the scene yourself — see `lighting.md` and
  `vsg::createHeadlight()` (`include/vsg/lighting/Light.h:49`).

## Composition examples

Distilled from `examples/lighting/vsgshadow/vsgshadow.cpp` (builds). Minimal end-to-end
wiring: a shadow-casting directional light plus tuned cascades.

Only the shadow-specific delta is shown; `scene` (a `Group`) and `camera` come from the
standard viewer setup in `references/examples/model-viewer.md`.

```cpp
// 1. choose a shadow technique (numShadowMaps = cascade count).
auto shadowSettings = vsg::HardShadows::create(/*numShadowMaps*/ 1);   // vsgshadow.cpp:314
// or vsg::SoftShadows::create(1, /*penumbraRadius*/ 0.05f);           // :308
// or vsg::PercentageCloserSoftShadows::create(1);                     // :301

// 2. a DirectionalLight that CASTS shadows — attach shadowSettings to the LIGHT.
auto sun = vsg::DirectionalLight::create();                           // :532
sun->color.set(1.0f, 1.0f, 1.0f);
sun->intensity = 0.95f;
sun->direction = vsg::dvec3(0.0, 0.0, -1.0);                          // :536
sun->shadowSettings = shadowSettings;                                 // :538 (public member)
scene->addChild(sun);

// 3. an AmbientLight so shadowed areas aren't pure black (never casts). :546-550
auto ambient = vsg::AmbientLight::create();
ambient->color.set(1.0f, 1.0f, 1.0f);
ambient->intensity = 0.05f;
scene->addChild(ambient);

// 4. View::create() defaults to RECORD_ALL, so shadow recording is ON (:626);
//    tune the cascades on the View's ViewDependentState.
auto view = vsg::View::create(camera, scene);                         // :626
view->viewDependentState->maxShadowDistance = 1e8;   // far cut-off   // :628
view->viewDependentState->shadowMapBias     = 0.005; // anti-acne bias// :629
view->viewDependentState->lambda            = 0.5;   // cascade split // :630

// 5. wire view -> RenderGraph -> CommandGraph as usual (see commandgraph.md), then:
viewer->compile();   // auto-sizes the shadow maps from the lights it finds. :654
```

Override the shadow settings for every light from the view (e.g. force HardShadows
regardless of what each light asked for):

```cpp
// matches any Light not explicitly keyed in the override map. vsgshadow.cpp:631
view->viewDependentState->shadowSettingsOverride[{}] = vsg::HardShadows::create(1);
```

PCSS / SoftShadows additionally require the matching shader define so the fragment shader
takes the right code path (the example sets these on a `ShaderCompileSettings`):

```cpp
auto shaderHints = vsg::ShaderCompileSettings::create();
shaderHints->defines.insert("VSG_SHADOWS_PCSS");   // for PCSS  — vsgshadow.cpp:300
// shaderHints->defines.insert("VSG_SHADOWS_SOFT"); // for SoftShadows — :307
// shaderHints->defines.insert("VSG_SHADOWS_HARD"); // for HardShadows — :313
```

## Common mistakes

- Assigning `shadowSettings` to a `PointLight` or `AmbientLight` and expecting shadows —
  only `DirectionalLight`/`SpotLight` cast (`examples/lighting/vsgshadow/vsgshadow.cpp:538,586`).
- Hand-wiring a `View` and forgetting to add any light: a hand-built `View` has no
  auto-headlight, so lit geometry renders black. Add lights or use
  `vsg::createHeadlight()` (`include/vsg/lighting/Light.h:49`).
- Setting `shadowSettings` after `viewer->compile()` — the maps are sized at compile time,
  so configure the lights first (`examples/lighting/vsgshadow/vsgshadow.cpp:538,654`).
- Tuning shadow bias/distance on the light — those knobs live on
  `view->viewDependentState`, not on `ShadowSettings` (`include/vsg/state/ViewDependentState.h:154-156`).
- Selecting Soft/PCSS without inserting the matching shader define
  (`VSG_SHADOWS_SOFT` / `VSG_SHADOWS_PCSS`) — the technique won't take effect
  (`examples/lighting/vsgshadow/vsgshadow.cpp:300,307`).

## Things to never invent

- There is **no** `View::maxNumShadowMaps` and no shadow-map capacity setter on `View` —
  `viewer->compile()` auto-sizes shadow maps; cascade count is `ShadowSettings::shadowMapCount`
  (`include/vsg/lighting/ShadowSettings.h:26`).
- There is **no** `Light::setShadowSettings()` — `shadowSettings` is a public member you
  assign directly (`include/vsg/lighting/Light.h:34`).
- Do not invent a `SoftShadows::setPenumbraRadius()`; `penumbraRadius` is a public member
  (and a ctor arg) (`include/vsg/lighting/SoftShadows.h:23,26`).
- `HardShadows` and `PercentageCloserSoftShadows` declare **no** extra public data members
  beyond the inherited `shadowMapCount` — do not invent quality/sample fields on them
  (`include/vsg/lighting/HardShadows.h:20-34`, `include/vsg/lighting/PercentageCloserSoftShadows.h:20-34`).
- There is no `ShadowSettings` member on `View` or scene nodes; shadows attach to a
  `Light`, with the only view-side hook being `ViewDependentState::shadowSettingsOverride`
  (`include/vsg/state/ViewDependentState.h:159`).
